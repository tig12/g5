Cura data

Gauquelin5 program handles a subset of the data published on cura.free.fr : only persons with known names (famous persons).
This gives 21 249 persons (A1 -> A6, D6, D10, E1, E3).
They form the departure point of g5 database ; Gauquelin data are imported first, then Müller and skeptics and other complete the database.

Restoration is still incomplete (eg miss some names in serie A, miss birth place in D6, timezone offset computation...), further work is needed ; see generated files.

Gauquelin unique id

Each person in g5 database has a unique id formed by its "slug", and several local ids, only unique within a source (see page on database).
Applied to Cura data, each person is at least related to 2 sources : one source called "cura" and one source related to its file of origin (ex "A1").
This gives at least 2 local ids for each person :
  • One id referring to its file of origin, using its field NUM of Cura file.
  • One id referring to "cura" source. This one is called the "Gauquelin unique id" (abbreviated GQID).
    It is built concatenating the id of its file of origin and its NUM within this file.
    Ex : for the first record of file A1 (Pierre Alard 1937-09-17), its GQID is A1-1.

For duplicate values (listed in several Cura files), the file where it was first published is retained to build its GQID - this corresponds to the alphabetical order of file names (A1 was published before A2 etc. - see Cura web site).

Example :
Louis Geraud 1902-07-06 is NUM 2054 in file A1 and NUM 1008 in file A2.
Its g5 unique id (slug) is geraud-louis-1902-07-06
Its local ids are :
Source Id
cura A1-2054
A1 2054
A2 1008
The GQID of this person is A1-2054.

Similarily a Müller unique id was defined.

Working on cura data

If you type : 
php run-g5.php cura
the program tells you the possible values of datafiles for cura dataset : 
Possible datafiles for dataset cura : all, A, A1, A2, A3, A4, A5, A6, D6, D10, E1, E3
  • all will operate on all cura files.
  • A will operate on all files of serie A.
  • Other values will operate on a single file.

The precise list of transformations depends on the concerned file, but for all files,
- the first transformation is raw2tmp,
- the second transformation is tweak2tmp
- the last transformation is tmp2db.

raw2tmp

For all cura datafiles, the first transformation is raw2tmp
It extracts data from html files located in data/raw/cura.free.fr and produces csv files located by default in data/tmp/cura (tmp dir can be modified in config.yml).

This is done for example with the command : 
php run-g5.php cura D6 raw2tmp
During this step, some corrections are applied to the data.
Some corrections are common to all files :
  • Country codes are converted to ISO 3166.
  • Dates and times are merged to be stored in ISO 8601 format YYYY-MM-DD HH:MM:SS.
    In this step, dates are not modified ; if needed, legal time restoration (for A files) is done in a separate step.
  • Longitudes and latitudes are expressed in decimal degrees.
  • Occupation codes are computed.
  • Family and given names are computed when possible.

tweak2tmp

This is done after raw2tmp, see page tweak2tmp.

Other transformations

Other transformations (between tweak2tmp and tmp2db) may modify the files located in data/tmp/cura.

Precise transformations are described in the following pages :
  • Serie A
  • File D6
  • File D10
  • Files E1 and E3

Export to csv

Done with 
php run-g5.php cura <datafile> export
where <datafile> can be any value understood by cura (A, A1 ... E3)

  • 212 persons appear twice
