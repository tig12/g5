Source

This page describes the Source data type of g5 database.
Sources are built by humans filling yaml files in 7-full/sources/.

YAML template

The structure of a source is given by src/model/Source.yml 
document:
  history: []

uid: 
id: 
type: 

### source stuff ###
source:
  parent:
  notes: |

### content ###
name: 
author: 
date: 
N: 
description: 
  
citations: []

publisher:

publications: []

Source uid

Like all entities of db5, a source is characteized by an uid, unique within g5 db.
A source uid reflects the path of the yaml file containing informations about the source :
Source with uid = muller402 source/web/newalch/muller402
is stored in data/7-full/source/web/newalch/muller402.yml

Source id


Sources have a supplementary identifier, named id. the id of a source is unique within g5 sources. It can be seen as a shortcut, a simpler way to refer to a source
Ex : booklets published by Arno Müller between 1991 and 1994
  • uid = source/booklet/Astro-Forschungs-Daten
  • id = AFD

ids are case-sensitive.

The associations between source ids and uids are maintained using file index/source/id-uid.txt.
Contains lines with <id> <space> <uid>
ex: 
AFD source/booklet/Astro-Forschungs-Daten
muller402 source/web/newalch/muller402

Source type

type : the type of the source.
Source type currently used :
  • article
  • book
  • booklet
  • file
  • organization
  • periodical
  • unpublished
  • web
Directory 7-full/source is organized by source type.
This has a consequence : sources related to a given topic are located in different directories on the disk.
For example, Arno Müller's experiment on 402 italian writers relate to these sources :
  • source/booklet/Astro-Forschungs-Daten
  • source/web/newalch/muller402
