Cura data

Gauquelin data published on cura.free.fr are available as html web pages.

Download cura data

The first step to work on the data is to download the html pages containing the data on your local machine.
You can use the script build/get-data, or download the files one by one.

The important thing is that the html files must be all stored in the same directory, specified by entry 1-cura-raw/ of config.yml.
They must be stored with the same name as in cura.free.fr (ex : 902gdA1y.html).

Issues

The problem of time

Here, "legal time" means "the time that can be read on the birth certificate".
In cura.free.fr, times are not presented in a uniform way. Cura files can be divided in two groups :
  • Files with all informations available to compute astrological charts
    • In files of series A, B, D9, F, NH the legal time is transformed, and a corresponding timezone offset is given.
    • In file D10, the legal time is given with the timezone offset.
  • Files with missing informations
    • In files E1 and E3, legal time is given with place name.
      Place name must be linked to a standard geographical frame in order to compute the timezone offset.
    • In file D6, longitude and latitude are given without place name and country code. Reverse geocoding is necessary to find the country and compute timezone offset.

I think the presentation done in series A, B, D9, F, NH is bad, because it mixes two distinct informations (legal time and timezone offset) in one data (the birth time).
This complicates the verification process : to check a birth hour on a civil registry, one first need to deal with the timezone offset problem.
So whenever possible, I restored the legal time with the corresponding timezone offset.
The concerned files contain in their notice Birthtime is converted to standard time, either for zone 0 (= GMT) or -1 (= CET)
