Cura data

Gauquelin data published on cura.free.fr are available as html web pages.

Download cura data

The first step to work on the data is to download the html pages containing the data on your local machine.
You can use the script get-data, or download the files one by one.

The important thing is that the html files must be all stored in the same directory, specified by entry 1-cura-raw/ of config.yml. They must be stored with the same name as in cura.free.fr (ex : 902gdA1y.html).

Issues

The problem of time

Here, "legal time" means "the time that can be read on the birth certificate".
In cura.free.fr, times are not stored in a uniform way.
In some files (like all the files of serie A), the legal time is transformed, and a corresponding timezone offset is given.
In some files (like D6, D10), the legal time is given with the timezone offset.

In both situations, the information presented permits to compute correctly the instant of birth.

But I think the presentation done in serie A is bad, because it mixes two distinct informations (legal time and timezone offset) in one data (the birth time).
This complicates the verification process : to check a birth hour on a civil registry, a transformation needs to be done first.
So whenever possible, I restored the legal time with the official timezone.
