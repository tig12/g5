Acts

Acts are the original documents, birth certificates (BC). These are the only reliable information sources, and every g5 data should be checked against a BC. Few birth times have been verified to resolve questions raised by g5 development.
This page describes the verification process and the integration of acts in g5.
Acts are stored in directory 8-acts/.

About french data

Currently, data are checked only for France. All acts since 1792 have times indicated. Scans of the original documents until ~1920 can be viewed online for free. This makes 5 generations, around 200 millions of acts (birth, death, mariage) with times (source filae.com).
Here lies potential data to test planetary heredity at large scale, but this is out of the scope of g5 program.

If you want to see original acts, type on a search engine for example "archives 45" (or maybe "Archives 45 France") to see the archives départementales du Loiret (Loiret is département n° 45).

Retrieval process

For each verification,
  • the image
the image is downloaded on local machine, and informations to easily go back to the online act are recorded in a yaml file expressed in a syntax which can be used to match g5 data by program.

The following informations are necessary :
(in parentheses, corresponding fieldname in g5 vocabulary).
  • Country (birth.place.cy)
  • Département (birth.place.c2)
  • City (birth.place.name)
  • Arrondissement (birth.place.c3), for Paris, Lyon, Marseille (arrondissement = part of a city).
    Important because without it, it's very difficult to find the registry.
  • Name of the registry
  • Page number where the birth certificate is located within the registry.
    Important because finding the right page may be a long process.
  • If possible the url directly pointing to the page. Some départements provide a way to copy the "permalink" of a document, which may be different from the url of the page displayed by the web browser.

Image download

This is systematically done during g5 development. Some départements provide a link to download current page, some don't. When they don't, image retrieval is done with screen capture. This can be particularily long because sometimes several captures are necessary to get the whole certificate with enough details. In this case, the image is reconstituted assembling the different parts.

Directory layout

                    └── galois-evariste-1811-10-25
                        ├── naissance.jpg
                        └── naissance.yml

Example

Acts syntax

The syntax used to transcribe the acts comes from an unpublished genealogy software. YAML files are written using french vocabulary, and are translated to a reference vocabulary (close to english) understood by code parsing the yaml files.
This syntax is adapted to act transcription and contains informations useless for g5 development.
