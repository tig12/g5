Output format

In all generated csv files, the first line contains field names ; other lines contain data.
The field separator used in csv file is ; (semicolon).
Character encoding is UTF-8.

Common fields

The following fields are common to several series :
Field name Comments
NUM Original NUM record number coming from cura.free.fr.
C_APP Indicates if the data was corrected in the review "Astro-Psychological Problems".
This field permits to express an information from cura.free.fr :
Also, in these series A (and for vol. D.10) I have added an exclamation point for the professionals whose birth data have been corrected from Astro-Psychological Problems, the Research Journal of Françoise, particularly in vols. 3.2, 3.3, 4.1, 4.2, 4.3, 5.1, 7.1, 7.2 and 10.2 (PG, 02-2009)
Can be 'Y' (if the record was corrected in APP) or empty string (if the record was not corrected in APP).
For corrected records, the exclamation point is removed from NUM.
NAME Person name when available.
If this field is present, it means that family name and given name have not been computed.
FAMILYNAME Family name, when it was possible to compute.
GIVENNAME Given name, when it was possible to compute.
PRO Profession code, see below.
When a person has more than one profession code, the codes are separated by "+". Ex : PO+WR means that the person is identified as a politician and a writer.
DATE Birth date ; ISO 8601 of this form : YYYY-MM-DD HH:MM:sHH:MM (timezone offset is included)
Example : 2017-05-03 09:30:00+02:00 :
Legal time is 09:30:00 and timezone offset is +2 hours.
PLACE Birth place name.
COU ISO 3166 country code, 2 letters format.
ADM2 Administrative division (département in France, state in USA) ; equivalent of COD in cura files.
GEOID geonames.org unique identifier of the place.
LG Longitude in decimal degrees.
LAT Latitude in decimal degrees.
