Output format

Here are the field names used in files of data/5-tmp/
In the csv files, the first line contains field names ; other lines contain data.
The field separator used in csv file is ; (semicolon).
Character encoding is UTF-8.
More details in /gauquelin5/download/README.

Common fields

Some csv files contain fields not listed here, in this case fields are documented in the README file of the csv directory.
Field name Comments
NUM Unique identifier of a record in a given context.
Corresponds to NUM in cura.free.fr files.
FNAME Family name, when it was possible to compute.
GNAME Given name, when it was possible to compute.
NAME Person name
If this field is present, it means that family name and given name have not been computed.
G Gender
F or M in general.
OCCU Occupation code
Corresponds to field PRO in cura files.
When a person has more than one occupation code, the codes are separated by "+". Ex : PO+WR means that the person is identified as a politician and a writer.
See below for a full list of codes.
Dates and places
If a file contains only birth data, DATE and PLACE are related to birth.
If a file contains birth and death data,
  • BDATE and BPLACE are related to birth.
  • DDATE and DPLACE are related to death.
DATE Birth date ; ISO 8601 of this form : YYYY-MM-DD HH:MM:sHH:MM (timezone offset is included ; s means "sign" + or -).
Example : 2017-05-03 09:30:00+02:00 :
Legal time is 09:30:00 and timezone offset is +2 hours.
For records with no birth time, format is YYYY-MM-DD
PLACE Birth place name.
BDATE Birth date ; same format at DATE
BPLACE Birth place name.
DDATE Death date ; same format at DATE
DPLACE Death place name.
Geographical information
CY country code ; ISO 3166, 2 letters format.
Corresponds to field COU in cura.free.fr files.
C2 Administrative division level 2
Means "département" in France ; means "state" in the USA.
Corresponds to field COD in cura.free.fr files.
Corresponds to ADM2 in geonames.org ("second-order administrative division").
GEOID geonames.org unique identifier of the place.
LG Longitude in decimal degrees.
LAT Latitude in decimal degrees.
Miscelaneous
C_APP Indicates if the data was corrected in the review "Astro-Psychological Problems".
This field permits to express an information from cura.free.fr :
Also, in these series A (and for vol. D.10) I have added an exclamation point for the professionals whose birth data have been corrected from Astro-Psychological Problems, the Research Journal of Françoise, particularly in vols. 3.2, 3.3, 4.1, 4.2, 4.3, 5.1, 7.1, 7.2 and 10.2 (PG, 02-2009)
Can be 'Y' (if the record was corrected in APP) or empty string (if the record was not corrected in APP).
For corrected records, the exclamation point is removed from NUM.

Occupation codes

I didn't find a convenient standard for occupation codes, so I built arbitrary codes.
Here is a complete list of occupations codes, used in all generated files.

2 letters are used for general categories (ex AR = artists)
3 letters are used for more precise occupations (ex MUS = musicians).

Note : these lists were generated from share/yaml/data/groups/gauquelin-occupations.yml by tools/occupation-codes.php.
