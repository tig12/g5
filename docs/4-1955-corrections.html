1955 corrections

This step is time-consuming : for each record of the original groups, compare the values of fields with values of cura data and note the original values when they differ.
This step is illustrated with the restoration of group "570 sportifs" from file A1.

Prepare edited files

Once the records of the original groups have been identified in previous step, the first thing to do is to generate the files which will be edited by humans. This is a two-step process :
  1. Generate files containing only the records of the original groups.
    This is done with command 
    php run-gauquelin5.php 1955 marked2generated
    This will extract information from files in 3-cura-marked/ to generate files in 4-1955-generated/.
    In config.yml, the setting 1955 - sort permits to specify which field is used to sort the generated file (obtaining a file as convenient as possible for human edition is important).
  2. To avoid accidental erasure of human modification, the files of 4-1955-generated/ must be copied to 5-1955-edited/
    For example : 
    cp data/4-1955-generated/570SPO.csv data/5-1955-edited/
At this stage, the file hierarchy looks like that : 
data
    ├── 1-cura-raw
    ├── 2-cura-csv
    ├── 3-cura-marked
    │   ├── A1.csv
    │   └── A2.csv
    ├── 4-1955-generated
    │   ├── 349SCI.csv

    │   └── 576MED.csv

Human corrections

This is the long part of the work : note the differences between Gauquelin 1955 book and cura data.
Once a file has been copied to 5-1955-edited/, columns where the corrections will be written are manually added.
These column are named with a prefix C_ (C for Correction).
  • GIVEN_C : Given name
  • FAMILY_C : Family name
  • HOUR_C : Hour HH:MM
  • DAY_C : Day YYYY-MM-DD
  • PLACE_C : Name of place (Exact spelling of geonames.org)
  • COD_C : COD in cura vocabulary = ADM2 in geonames = département for France
  • COU_C : ISO 3166 country code
  • PRO_C : Profession code
  • NOTES_C : Free notes
The column are filled only when the values read in the book differ from the values retrieved from cura file.
This step is easier when two persons work together : one reads the book, the other writes the corrections

Some records were present in Gauquelin book and not in cura file ; in this case, the field ORIGIN is noted G55.

Generate results

These edited files are used to generate files in three directories :
  • 9-1955-original
    If a field has a corrected value, it will be retained.
    The not-corrected fields will be taken from corresponding cura file in directory 2-cura-csv.
    Generation done with the command 
    php run-gauquelin5.php 1955 generateOriginal
  • 9-1955-corrected
    Same as generation of 9-1955-original, but the corrected values of DAY_C and HOUR_C are ignored.
    This is done under the hypothesis that cura data contain less errors than Gauquelin 1955 book.
    This hypothesis must be checked with careful tests
    Generation done with the command 
    php run-gauquelin5.php 1955 generateCorrected
  • 9-cura-corrected
    Same as generation of 9-1955-corrected, but for all the records of files located in 2-cura-csv.
    Generation done with the command 
    php run-gauquelin5.php A1 generateCorrected
