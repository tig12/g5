7-full

Data about famous persons need to be merged to be related to common identifiers like VIAF, ISNI, BNF, Wikidata...
This is done in a directory, by default data/7-full/.
7-full forms a database, stored in yaml files instead of using a relational or graph database.
  • Person files
  • g5 integration
  • WIP
Current code converts raw files to csv, stored in 5-tmp/.
Conversion of this code to use 7-full/ instead has begun (starting with cura A).

In 7-full/, each person is stored in one yaml file.
Building csv file is now done in two steps :
  • Import raw files to yaml in 7-full/
  • Generate csv in 9-output/ from yaml files.

7-full/ currently looks like that : 
7-full
    ├── groups
    │   ├── history
    │   └── new
    ├── lost
    ├── persons
    ├── pilot
    └── sources
  • groups/ reproduce the hierarchy of 9-output/.
    groups/ contains text files
    ex: 7-full/groups/datasets/cura/A2.txt 
    persons/1873/12/15/joseph-abadie
persons/1876/08/12/jules-abadie
persons/1864/03/10/jacques-abelous
...

    Each line is the uid of a person member of the groups, see data model.
    Generated by program
  • lost/ : logs of executions needing human inspection.
    Generated by program and cleaned by humans.
  • persons/ contains one yaml file per person.
    Generated by program and edited by humans
  • pilot/ to assist the merge process.
    Edited by humans
  • sources/
    Information sources
    Edited by humans
pilot/ and sources/ are filled by humans and versioned.

The database is composed by
  • Persons

Person files

Each person stored in g5 database has one directory
Organized chronologically : 
data
   └── 7-full
      └── persons
         └── 1937
            └── 09
               └── 17
                  └── alard-pierre
                     └── alard-pierre-1937-09-17.yml
(This may change, with one directory per person, instead of one file)

Example data/7-full/persons/1937/09/17/alard-pierre/alard-pierre-1937-09-17.yml (the first record of file A1) : A Person is an associative array using a syntax described in data model.
It contains 4 administrative fields :
  • ids : Associative array
    keys = source id
    values = ids of this entity within this data source.
  • sources
  • history Regular array
    Each entry is an assoc. array of 4 elements : date, command, source, values.
    values contains the new values of the fields updated during a command execution.
    Commands use this field to log their execution.
    It permits to trace each transformation operated on a data.
  • raw Associative array
    keys : source id
    values : raw values from the file at the origin of an update.
Other fields are the person's characteristics (in the same vocabulary as history values).

An empty person is created from a template YAML file : src/model/Person.yml. Contains the possible fields of a person with empty values.

g5 integration

Previous commands don't change, except raw2csv, replaced by raw2full. 
php run-g5.php cura A1 raw2full small
--- Importing file A1 ---
Corrections from 1955 book : 71
names : nb match = 1969 / 2087 (94.35 %)
    nb NOT match = 118 (5.65 %)
Strored 2087 records
Then generate csv in 9-output : 
php run-g5.php cura A2 export
Exported groups/datasets/cura/A2 to data/9-output/datasets/cura/A2.csv

7-full is managed by 3 classes of model : Full, Person, Group. 
class Full{
    const SEP = '/';
    public static $DIR; // Path pointing to 7-full/
}

class Person{
    public $uid = '';
    public $data = [];
    public static function new($uid): Person
    public static function newEmpty(): Person
    public function uid(): string
    public function slug(): string
    // file system
    public function path($full=true): string 
    public function load()
    public function save()
    // fields
    public function addId($source, $id)
    public function addSource($source)
    public function addOccu($occu)
    public function setOrigin($source, $data)
    public function addHistory($command, $source, $data)
    public function update($replace)
}

class Group{
    public $uid;

    // file system
    public function path($full=true): string 
    public function load()
    public function save()
    public function exportCsv($file, $csvFields, $map=[], $fmap=[])
    // fields
    public function add($entry)
}
