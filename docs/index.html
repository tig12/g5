Gauquelin5

This program operates on data used to perform statistical tests on astrology since 1955.
Current code works on data of famous people (serie A, files D6, D10, E1, E3)
The purpose is to :
  1. Provide corrected versions of existing datasets,
  2. Restore the groups used in the historical tests when possible,
  3. Link these data with other datasets, like wikidata.org.
  4. Build new datasets, more complete and more precise, using web data available today.
  5. Build new eminence indicators.

If it's possible to reach a consensus to build a consensus on building datasets and eminence indicators, then it should be possible to reproduce the tests and observe if there are statistical anomalies or not.

The program generates dowloadable partially restored data.
Current code contains steps to go towards goals 1 and 2, and preparatory code for steps 3 and 5.
Raw input data come from
  • cura.free.fr C.U.R.A Gauquelin archives version 5
  • Gauquelin 1855 paper book "Les hommes et les astres"
  • newalchemypress.com

Why Gauquelin5 ?

There are several reasons to care about the data used to perform statistical tests on astrology :
  • Scientific interest
    The "mars effect" polemic lasted for several decades and is not finished. It ended with the Ertel vs Nienhuys arguments, and they reached radically opposed opinions. The question to know if these statistical anomalies exist out of human bias is still open.
    Data collected by Gauquelin and others can serve as a basis to build new groups and execute new tests and see if a consensus can emerge.
  • Historical interest
    Gauquelin and related data are the main material used to perform the astrological tests. So anyone who wants to study the details of this aspect of recent history needs the data.
    As this is a highly polemic topic, the data permit to check the affirmations of the historical protagonists.
  • Technical interest
    These data are interesting because they contain birth time. And birth times are generally neglected in the semantic web.
    Take for example wikidata.org : birth dates are set to 00:00:00. There is no way to make the difference between "this is a date with no time information" and "this is a date with time = 00:00:00" (maybe there is a way, but I didn't find it).
    Storing birth times also obliges to find a way to do it correctly. This problem is detailed in the page about time.
    Introducing birth times in the semantic web is not only useful for astrology ; it can be interesting for demography, sociology etc.

Corrected version of existing datasets

The definition of "good data" used in this program is :
  • Expressed in open and standard formats.
    • Encoded in UTF-8. The program does not handle other character encodings. Raw data encoded in other encodings still in use in some systems must be converted before being processed. All generated data use UTF-8.
    • Use only open format. The program mainly uses csv (txt, json and yaml are also used).
    • Express dates in a standard format. The program uses ISO 8601. This choice is not completely satisfactory because ISO is not fully available for free, and does not permit to express timezone offsets with a precision of one second (see the page about time).
    • Express countries in a standard format. The program uses ISO 3166 2-letter codes.
    • Use a standard list of profession codes among all the generated files. This is not done as a standard list was not found. The program uses a home-made list of code (which is not satisfactory).
  • Precise and identifiable without ambiguity.
    • Different forms of a person's name are sometimes needed :
      • Exact spelling of family and given names used in the civil registries
      • Name commonly used to designate the person.
      • Spouse name for woman.
      • Nickname or stage name.
      The program only deals with the common name.
    • Specify the gender of a person. The program does not generate this information yet.
    • Birth date and times should be expressed exactly as indicated in civil registries.
    • Timezone offset may complement this information (see the page about time). This is in general handled by the program.
    • Places should be related to a standard. The program uses geonames.org.
    • Persons should be related to at least one standard id. This is not currently not done by the program, but preparatory code has been written to link the data to wikidata.org.

A complete correction of data would need an exhaustive check with civil registries ; this is now possible at least for persons born in France before world war 1, because most french civil registries can be freely consulted online - but it's a very long and teddious process and hasn't been done.

Restoration of the original data sets

Ideally, a restoration process would lead to two versions of the data sets used in the historical tests : an exact historical version used by actors (errors included) and a corrected version.
The program currently only operates on the groups published in 1955 by Michel Gauquelin.
Restoration of the exact historical version has only been done for one group (570 sportifs), using the original book of Michel Gauquelin, "L'influence des astres", published in 1955.

Current results

Current code improves existing datasets, but is far from producing "good data", as you can see in the pages about Cura results and Gauquelin 1955 results.

This documenation is written in a globish as close as possible to english, but uses french conventions for capitalization, spaces and large numbers :
  • Names of months, weekdays, languages etc. are not capitalized.
  • The characters ; : ? ! are preceeded and followed by a white space.
  • White space is used as separator for large numbers (for example one million is written 1 000 000 instead of 1,000,000).
