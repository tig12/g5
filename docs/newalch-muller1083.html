Müller 1083
(1083 physicists)

This page deals with the file 5muller_medics.txt downloaded from newalchemypress.com.
This file is interesting because it contains good quality data. In particular, person and place names are generally more precise than Cura file A2.
It contains records from Cura files A2 and E3.
  • Integration to g5
    • Generate csv
    • Fix cura A2 names

Integration to g5

Very few work has been done on this file for the moment.
It could be used to
  • Check legal time restoration in A2.
  • Better place names in A2 (which would then permit better matching to geonames).
  • And it brings new persons not present in Cura files.

Generate csv

The first step is to generate the file 5-newalch-csv/1083MED.csv : 
php run-g5.php newalch muller1083 raw2csv

Fix cura A2 names

php run-g5.php cura A2 muller2csv
WRONG USAGE - muller2csv needs one parameter. Can be :
  update : modify cura file
  echo : echo detailed names
ex : php run-g5.php cura A2 muller2csv update

php run-g5.php cura A2 muller2csv update
Nb missing names in cura A2 : 207
Müller 1083 contains : 739 lines from A2
Nb corrections : 12
Only 12, the result is quite disapointing.

Looking at the file

NOTE : the rest of this page contains notes resulting from a first superficial look at this file. The code was written to try to understand the content of the file.
I don't understand all the columns, so consider the informations about the file as suppositions, there are probably mistakes.

With class src/transform/newalch/muller1083/look.php : 
php run-g5.php newalch muller1083 look 
PARAMETER MISSING in g5\transform\newalch\muller1083\look
Possible values for parameter : muller1083
For example : 
php run-g5.php newalch muller1083 look sample

Column SAMPLE

This column seems to indicate the origin of the record.
Here are the codes of this column and the number of associated records. 
php run-g5.php newalch muller1083 look sample
[GAUQ_NUR] => 168
[MUERGAUQ] => 612
[MUERGAUQ-d] => 79
[MUER_NUR] => 224
Remarks from newalchemypress.com give indications :
Only for those in Michel's original group born in Paris (168) was Müller unable to re-check their birth data (...) This new total of Academie members had 224 not in MG's original sample.
So GAUQ_NUR should correspond to people born in Paris, but there is a contradiction, as some records marked with GAUQ_NUR are not born in Paris.

MUER_NUR is easier to understand and corresponds to Newalchemypress remark : all records marked with MUER_NUR have an empty value in column GNR.

Column GNR

Maybe means "Gauquelin Number Record".
Values starting by SA2 correspond to Cura file A2 (SA may mean "Serie A") ; for example SA232 corresponds to record NUM 32 in file A2.
Values starting by ND1 correspond to Cura file E1 (ND1 may mean "New Data 1") ; for example ND1204 corresponds to record NUM 204 in file E1.

This command counts the records associated to Gauquelin files : 
php run-g5.php newalch muller1083 look gnr
A2 	: 765
E1 	: 94
EMPTY 	: 224
Sum = 1083
