Expressing time

This page deals with a small but important detail : the way to express times.
In this page, "legal time" means "the time that can be read on the birth certificate".

In some files (for example series series A, B, D9, F, NH of cura.free.fr), birth time is provided like that : 

YEA	MON	DAY	PRO	NUM	COU	H	MN	SEC	TZ	LAT	LON	COD	CITY		
(...)
1862	6	3	C	876	F	6	51	16	0	48N50	2E11	92	ASNIERES
It means that birth time is 06:51:16 and that timezone offset is 0.
The value 06:51:16 is the result of a computation : legal time + timezone offset.
This is coherent with the notice found in the concerned files : Birthtime is converted to standard time, either for zone 0 (= GMT) or -1 (= CET) I think that expressing times this way is a very bad idea because it mixes two distinct informations :
  • The legal time, as it can be read in civil registries.
  • The timezone offset.
It's a problem, because if one wants to check the data, it's impossible to check legal time without worrying about timezone offset.
Doing like that generates a loss of information. If the data is not correct, does it come from an error on legal time or on timezone offset ?

Using ISO 8601 format, this date would be expressed 1862-06-03 06:51:16+00:00
Restoration process computes birth time to 7 AM and timezone offset to 8' 44''.
So this date should be expressed 1862-06-03 07:00-00:08:44
